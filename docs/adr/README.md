# ADR (Architecture Decision Records)

このディレクトリではアーキテクチャ上の意思決定を記録します。

## 管理対象

番号付きの Markdown ファイルで管理します。

例: `0001-spring-boot.md`

## フォーマット

各 ADR には以下を記述します。

* タイトル
* ステータス（提案中 / 採用 / 却下 / 廃止）
* コンテキスト（なぜこの決定が必要だったか）
* 決定内容
* 結果（トレードオフ・影響）

## 方針

一度記録した ADR は原則変更しません。

決定を覆す場合は新しい ADR を作成し、古い ADR のステータスを「廃止」に更新します。
