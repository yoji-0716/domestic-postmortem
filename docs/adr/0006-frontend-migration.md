# ADR 0006: フロントエンドの段階的移行戦略

## ステータス
承認済み

## コンテキスト
最終的にはReactまたはVueによるSPAへの移行を想定しているが、
初期フェーズではドメイン設計の検証を優先し、フロントの実装コストを抑えたい。

## 決定
フロントエンドをThymeleaf（SSR）からReact/Vue（SPA）へ段階的に移行する。

### フェーズ1（現在）
- Thymeleaf + Spring MVC によるSSR実装
- ドメインロジックとUIの動作検証を優先

### フェーズ2（移行時）
- Spring MVCのControllerをREST API（@RestController）に切り替える
- フロントエンドをReactまたはVueで再実装する
- React/Vueのどちらを採用するかはフェーズ移行時に別ADRで決定する

## 理由
- 初期から SPAで実装するとフロントとバックの両方の設計コストが高い
- ThymeleafとSpring MVCの疎結合を保つことで、移行時の影響を最小化できる
- ドメイン層・アプリケーション層はフェーズ1・2で変更不要

## 結果
- フェーズ1ではControllerからViewにModelを渡すSSR構成とする
- ドメイン層・アプリケーション層はフレームワーク非依存を徹底し、フェーズ2でも再利用する
