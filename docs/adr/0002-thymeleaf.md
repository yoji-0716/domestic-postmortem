# ADR 0002: Thymeleaf の採用（暫定）

## ステータス
承認済み（将来的にReact/Vueへ移行予定 → ADR 0006参照）

## コンテキスト
フロントエンドの実装方針を決定する必要があった。
最終的にはReactまたはVueへの移行を想定しているが、初期フェーズでは開発速度を優先する。

## 決定
初期フェーズはThymeleafでHTMLを実装する。

## 理由
- Spring MVCとの統合が容易でセットアップコストが低い
- 初期フェーズではドメイン設計の検証を優先するため、フロントの実装コストを最小化したい
- 段階的なフロントエンド移行の足がかりとして機能する

## 結果
- 初期フェーズはThymeleaf + Spring MVCでSSRとして実装する
- React/Vue移行時にSpring MVCをREST APIに切り替える（ADR 0006参照）
