# 直近のプラン

## 次のセッション

### テスト設計の完成
AI実装エージェントが堅牢なコードを生成するために必要なテスト方針を定義する。

- ユニットテスト：ドメインエンティティ・ユースケースの振る舞いテスト
- 統合テスト：リポジトリ（H2）を使ったDB操作の検証
- 受け入れテスト：各UCの受け入れ条件（`acceptance-criteria.md`）に基づくシナリオテスト

---

## フェーズ1：実装（Thymeleaf）

設計書が揃い次第、AIによる実装を開始する。

### 実装順序（推奨）

1. **プロジェクト基盤**
   - DDDライクなパッケージ構成の骨格を作成
   - DB接続・JPA設定・マスタデータ投入（家族メンバー・カテゴリ）

2. **UC1: 家事・育児を記録する**（最もコアな機能）

3. **週次バッチ**（UC2/UC3の前提）
   - `WeeklySummaryScheduler` による自動集計

4. **UC2 + UC3: 週次実績・負荷バランス確認**

5. **UC6: カテゴリ重み設定**（UC3の計算に必要）

6. **UC4 + UC5: 振り返り・改善アクション**

### 完了条件
- 受け入れ条件（`acceptance-criteria.md`）の全チェックボックスが満たされること
