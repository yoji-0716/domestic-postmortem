# CLAUDE.md — プロジェクト作業ガイド

## プロジェクトの方針

このプロジェクトは **ドメイン設計に焦点を当て、実装はAIに委ねる** というアプローチを取る。

### 目標
- ドメインモデル・設計書・テスト設計を人間が丁寧に固める
- その設計をインプットとして、AIが素早く堅牢な実装を行う

### 優先すること
1. ドメインの正確な理解と表現
2. 設計書の完成度（ユースケース・ER図・アーキテクチャ等）
3. テスト設計の明確化

### AIへの期待役割
- 設計書が揃った段階でのコード実装
- 設計の意図を損なわない堅牢なコード生成

## ステアリングフォルダ（`.steering/`）

プロジェクトルート直下に `.steering/` フォルダが存在する。

### 構造

```
.steering/
  YYYY-MM-DD/        # その日の作業ディレクトリ（日付ごとに作成）
    <フェーズファイル>.md  # その日の作業を管理するフェーズファイル
```

### 概念

- **日付ディレクトリ**：作業日ごとに `YYYY-MM-DD` 形式のディレクトリを作成する。
- **フェーズファイル**：各日付ディレクトリ内に配置される `.md` ファイル。その日の作業フェーズや進捗を管理する。内容はそのときのタスクによって異なる。
